Homebrew CN 镜像安装

专为国内用户打造的 Homebrew 一键安装脚本，智能选择最佳镜像源，解决下载慢、安装失败等问题。

立即开始

复制以下命令到终端（Terminal）运行：

安装命令

$/bin/zsh -c "$(curl -fsSL https://brew-cn.mintimate.cn/install)"

卸载命令

$/bin/zsh -c "$(curl -fsSL https://brew-cn.mintimate.cn/install)" -- --uninstall

自动适配 macOS (Intel/Apple Silicon) · 脚本运行后可交互选择镜像源

核心特性
多镜像源支持 内置中科大 USTC、阿里云、清华 TUNA 等国内优质镜像源，支持一键切换。
全架构兼容 完美支持 macOS Intel (x86_64) 和 Apple Silicon (M1/M2/M3/M4) 芯片。
Shell 自动配置 自动识别 Zsh/Bash，智能写入环境变量配置，无需手动编辑配置文件。
安全备份 修改任何系统配置文件前都会自动创建备份，确保系统安全，随时可回滚。
安装流程
运行脚本

复制命令并在终端执行，脚本会自动检测系统环境。

选择镜像源

根据提示选择下载源（推荐中科大或阿里云），输入对应数字回车。

等待安装

脚本自动下载并配置 Homebrew，通常只需 1-3 分钟。

生效配置

安装完成后，执行 source ~/.zshrc 或重启终端即可使用。

自动化配置详情

脚本会自动配置以下环境变量，确保 brew install 速度飞快：

  • HOMEBREW_BREW_GIT_REMOTE brew 核心仓库镜像
  • HOMEBREW_CORE_GIT_REMOTE 核心软件仓库镜像
  • HOMEBREW_BOTTLE_DOMAIN 二进制预编译包镜像
  • HOMEBREW_API_DOMAIN Homebrew API 镜像
常见问题解答
安装后提示 "command not found: brew"
这是因为环境变量未生效。请执行 source ~/.zshrc (或 ~/.bash_profile)，或者直接关闭当前终端窗口并重新打开一个新的。
brew update 报错或卡住
可能是 git 仓库状态异常。尝试运行 brew update-reset 重置仓库，或者重新运行本安装脚本选择「重置配置」。
如何更换其他镜像源？
无需卸载，直接重新运行安装脚本，在菜单中选择新的镜像源即可。脚本会自动覆盖旧的配置。
需要安装 Xcode Command Line Tools 吗？
是的，这是 Homebrew 的必要依赖。脚本会自动检测，如果未安装会弹出系统提示，点击「安装」即可。
